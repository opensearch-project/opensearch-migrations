# Stakeholders

## Search Relevance Engineer

Focuses on search quality, schema design, and query behavior. Cares about field types, analyzers, tokenizers, synonym handling, scoring (BM25, LTR), Query DSL translation, and precision/recall outcomes. Expects deep technical detail on schema mappings and query conversion.

## DevOps / Platform Engineer

Focuses on infrastructure, deployment, and operations. Cares about cluster sizing, shard strategy, JVM heap, node roles, index lifecycle, monitoring, and cost. Expects guidance on architecture decisions and operational best practices for Amazon OpenSearch Service.

## Business Stakeholder

Focuses on project delivery, cost, and resource planning. Cares about total migration cost (infrastructure spend and engineering hours), timeline, milestones, risks that could delay the project, and budget impact. Has little interest in technical implementation details — wants findings translated into effort estimates, cost line items, and go/no-go decisions. Expects plain-language summaries, not schema JSON or Query DSL. When blockers are found, wants to know the business impact and estimated resolution effort, not the technical root cause.
