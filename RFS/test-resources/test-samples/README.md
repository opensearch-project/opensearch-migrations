# Test Sample Files

Expected-output fixtures for document and metadata extraction tests.

## How These Files Were Generated

Each JSON file was auto-generated by `ReferenceDocumentExtractionTest` in the `SnapshotReader` module.
The test reads documents from pre-built snapshot fixtures (in `../snapshots/`) using the Lucene
reader pipeline, serializes the output as pretty-printed JSON, and writes it here.

**To regenerate:** delete the sample file and re-run the corresponding test. The test detects the
missing file and writes the current extraction output as the new expected baseline.

## File Naming Convention

- `es56-docs.json` — Documents extracted from ES 5.6 snapshot
- `es68-docs.json` — Documents extracted from ES 6.8 snapshot
- `es68-merged-docs.json` — Documents from ES 6.8 snapshot with merged segments
- `es710-wsoft-docs.json` — Documents from ES 7.10 snapshot with soft deletes enabled
- `es710-wosoft-docs.json` — Documents from ES 7.10 snapshot without soft deletes
- `es56-global.json` / `es68-global.json` / `es710-global.json` — Global metadata
- `es56-index.json` / `es68-index.json` — Index metadata
- `es710-bwc-index.json` / `es710-fwc-index.json` — BWC/FWC index metadata

## Usage

These files are referenced by:
- `ReferenceDocumentExtractionTest` (SnapshotReader) — validates document extraction stability
- `TargetWriteEndToEndTest` (DocumentsFromSnapshotMigration) — validates writing extracted docs to target clusters
